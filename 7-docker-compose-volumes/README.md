# Module 6 - Docker Compose: Volumes

**Goal**: Learn how volumes persist data beyond the container lifecycle.

## Background

Volumes are the preferred mechanism for persisting data generated by Docker containers. Unlike container filesystem, volumes persist even when containers are stopped or removed.

## Steps

1. Examine the `docker-compose.yml` file:
   - A named volume `redis_data` is mounted to Redis at `/data`
   - Redis will persist its data to this volume

2. Build and start the services:
   ```shell
   docker compose up --build -d
   ```

3. Visit http://localhost:4000/counter multiple times to increment the counter:
   ```shell
   curl http://localhost:4000/counter
   ```
   Note the `page_views` value - it should be increasing.

4. **Stop the containers without removing volumes**:
   ```shell
   docker compose down
   ```

5. **Restart the containers**:
   ```shell
   docker compose up -d
   ```

6. **Check the counter** - it should still be there:
   ```shell
   curl http://localhost:4000/counter
   ```

7. **Now remove volumes and restart**:
   ```shell
   docker compose down -v
   docker compose up -d
   ```

8. **Check the counter again** - it should be reset to 1:
   ```shell
   curl http://localhost:4000/counter
   ```

## Inspect Volumes

List all volumes:
```shell
docker volume ls
```

Inspect the specific volume:
```shell
docker volume inspect 6-docker-compose-volumes_redis_data
```

## Key Concepts

- **Named Volumes**: Persist data independently from containers
- **Data Persistence**: Data survives `docker compose down` but not `docker compose down -v`
- **Host-to-Container Binding**: Bind mounts map host directories to containers (use `volumes: ./local:/container/path`)
- **Volume Drivers**: Docker supports various volume drivers for different backends

## Cleanup

```shell
docker compose down -v
```
